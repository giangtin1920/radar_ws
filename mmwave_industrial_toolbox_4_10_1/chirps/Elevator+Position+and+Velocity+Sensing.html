Elevator Position and Velocity Sensing

Introduction

This use case provides the sensing configuration for applications related to Elevator position and velocity determination. Commonly used solutions today use a combination of rotary encoders tied to motors which do not scale well past 200m shafts and their cable stretching impacts the accuracy of these encoders. mmWave Radar is well suited for this kind of application given its ability to contactlessly and accurately measure long distances and speeds with immunity to environmental factors such as dust, low/no light, smoke etc. which may be present in the elevator shaft in emergency situations.

Testing and Results

This chirp configuration needs a dedicated processing chain to work and it cannot work with OOB Demo processing chain. However, this chirp configuration has been tested to confirm that it correctly configures the Radar Front-End for the specified sensing configuration i.e. it doesn't generate any mmWaveLink API errors. This is done by loading the chirp configuration in the OOB Demo Configuration GUI (using the OOB Demo Configuration File provided in Step 3).
